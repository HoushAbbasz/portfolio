I applied margin-top to "#product-overview" section to give more space between the nav bar and the slogan. The slogan is now positioned slightly lower than it originally was.

I applied a z-index of 10 to ".package__badge", the badge now overlaps the header when scrolling. This occurs because the badge's z-index is now higher than the header's z-index (since the nav bar is nested within the header, they have the same stack context).

I fixed the stacking order by establishing a new stacking context and giving the header the highest z-index, which makes it always appear on top of any other elements on the page.

CSS Udemy Reflection : Assignment 3, Positioning Elements with CSS

Through this assignment, I learned how CSS positioning and stacking contexts affect layout and element layering on a webpage.

I began by examining the "#product-overview" section on the uHost page and adjusting its spacing using the margin-top property to create more separation between the navigation bar and the slogan. This helped me understand how margins influence spacing between two elements.

Next, I changed the positioning of the "#product-overview h1" element to position: absolute and used the bottom and left CSS properties with pixel values to reposition the slogan within its container. I then replaced the pixel values with percentage values, to ensue that the slogan would look attractive on all screen sizes.

On the Packages page, I explored how z-index and stacking contexts control the visual layering of elements. After applying a higher z-index value to the .package__badge, I observed that it overlapped the nav bar while scrolling. To resolve this issue, I established a new stacking context and assigned the header (which contains the nav bar) a higher z-index, ensuring it consistently appeared above other elements.

Overall, this assignment helped me reinforce my CSS expertise while also expanding my knowledge in regards to the stacking context.
